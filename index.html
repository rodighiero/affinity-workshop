A hands-on lab for both tools on this site: three exercises reading a single document on the Text page, then three building a similarity map on the Network. Work through them in order. Everything runs live in your browser; nothing is uploaded.

The Text page is a microscope, the Network a telescope. These exercises teach you to read what each shows — and to notice where it is wrong.

Bring your own material: one text to analyse, and a small set of texts (a dozen is plenty) to map — your own, a group's, or the built-in sample. Everything runs in the browser and assumes English: the readability, sentiment, keyword, and embedding logic all degrade on other languages.

1

Read your text as data

MeasurementsVocabulary

TaskAnalyze your text, then describe it in two sentences using only numbers — word count, average sentence length, lexical diversity, reading time.

LearnA document has a measurable shape before it has meaning. This is the vocabulary for everything that follows.

2

Find the keywords a network would use

Keywords

TaskCompare the most frequent and most distinctive keyword lists. Write down the five words you would choose to represent this text among others.

LearnDistinctiveness, not frequency, gives a text its identity — the same intuition behind the affinity map, which discards the terms everyone shares.

3

Two texts, one lens

All sections

TaskAnalyze two texts — yours and a neighbour's, or your introduction and your conclusion. Which sections separate them most: grammar, keywords, sentiment?

LearnComparison is where analysis becomes mapping — the conceptual seed of the whole network exercise. From here on, move to the Network.

4

Build your first network

Build networkStats overlay

TaskDrop a set of text files — or use the built-in sample — and click Build network. Before touching a slider, read the stats overlay: how many files, edges, clusters, and unconnected nodes? Note the strongest and average similarity.

LearnA network is a set of relationships made visible. The raw map, before any tuning, is the ground truth you will be reshaping.

5

Where “similar enough” begins

Threshold

TaskDrag the Threshold slowly from low to high. Watch edges vanish and nodes go unconnected. Find the value where the network first breaks into separate clusters — that number is your definition of “similar enough.”

LearnA link is a decision, not a fact. The threshold is where you draw the line between related and unrelated — and it changes what the map claims.

6

Read the clusters against the texts

ClustersElevation map

TaskTurn on the Elevation map so groups rise like high ground. Name each cluster and the node that sits alone, then open the texts behind one cluster. Find one grouping the machine got right and one you would contest.

LearnSimilarity is computed from words, not sense. Reading the clusters back against the documents is where interpretation returns to the map.
